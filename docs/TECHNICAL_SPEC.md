# Техническое задание: Marine Track MVP

## 1. Назначение

Marine Track MVP предназначен для автоматического определения положения, курса и скорости судов по открытым спутниковым данным. Базовая задача первого этапа — не полная классификация судов, а надежный конвейер:

1. найти подходящие сцены по AOI и времени;
2. загрузить или подготовить сцену;
3. обнаружить кандидаты судов;
4. обнаружить кильватерный след, если он выражен;
5. рассчитать положение и направление;
6. рассчитать скорость только валидным методом;
7. сохранить результат и параметры валидации.

## 2. Приоритет источников

### Sentinel-1 SAR

Основной источник MVP. Используется для круглосуточной и всепогодной детекции.

Fallback-порядок:

1. `asf` — NASA Alaska Satellite Facility, пакет `asf_search`.
2. `copernicus_cdse` — Copernicus Data Space Ecosystem STAC/OData.
3. `planetary_computer` — STAC-каталог Microsoft Planetary Computer.
4. `earthsearch` — STAC-каталог Element84, если коллекция доступна.

### Sentinel-2 MSI

Второй источник. Используется только при приемлемой облачности и дневной съемке.

Fallback-порядок:

1. `copernicus_cdse`.
2. `planetary_computer`.
3. `sentinelhub`.
4. `earthsearch`.

### Вспомогательные данные

- Copernicus Marine: течения, волны, ветер, SST.
- AIS: NOAA MarineCadastre, Global Fishing Watch, локальный приемник AIS.

## 3. Функциональные требования

### FR-1. Конфигурация запуска

Система принимает:

- AOI в GeoJSON;
- время `from/to` в UTC;
- сенсор: `auto`, `sentinel1`, `sentinel2`;
- выходной каталог;
- параметры облачности, поляризации, режима съемки.

### FR-2. Поиск сцен

Система должна пробовать провайдеры по порядку. Если провайдер недоступен или вернул пустой результат, включается следующий.

### FR-3. Предобработка

Sentinel-1:

- выбор поляризации;
- вырезка AOI;
- маска суши/берега;
- подавление спекл-шума;
- нормализация интенсивности.

Sentinel-2:

- загрузка B02/B03/B04/B08;
- cloud mask;
- water mask;
- ресемплинг к 10 м;
- локальный контраст.

### FR-4. Детекция судов

MVP-методы:

- SAR: адаптивный порог/CFAR, connected components, фильтр по площади и форме.
- Optical: порог по яркости/контрасту, water/cloud mask, connected components.

ML-детектор не обязателен для первого MVP.

### FR-5. Детекция следа

Для каждого кандидата берется окно 1–5 км.

Методы:

- локальное контрастирование;
- edge/ridge enhancement;
- Hough/Radon transform;
- выделение центральной оси турбулентного следа;
- выделение Kelvin arms, если они видны.

### FR-6. Положение

Приоритет методов:

1. centroid корпуса;
2. вершина V-shaped wake;
3. вероятный сектор по оси следа, если корпус не виден.

### FR-7. Курс

Приоритет методов:

1. wake axis;
2. ориентация корпуса;
3. Sentinel-2 inter-band displacement;
4. AIS только для проверки.

Если есть только вытянутый корпус без следа, фиксируется неоднозначность 180°.

### FR-8. Скорость

Скорость считается только при наличии физически валидного признака.

Методы MVP:

1. `sentinel2_interband_displacement`: смещение судна между каналами Sentinel-2.
2. `kelvin_wavelength`: скорость через длину волны в Kelvin wake.
3. `sar_offset_experimental`: только как экспериментальный флаг, не как основной метод.

Если метод неприменим, выводится `speed_method = not_estimated`.

### FR-9. Валидация

Виды валидации:

- AIS: интерполяция трека к моменту снимка, ошибки положения/курса/скорости.
- Физическая: проверка диапазонов скоростей, геометрии следа, согласованности направления.
- Кросс-сенсорная: согласование Sentinel-1 и Sentinel-2, если есть близкие сцены.

## 4. Выходные данные

Обязательные:

- GeoJSON с детекциями;
- Parquet/CSV для аналитики;
- quicklook PNG;
- HTML-отчет по сцене;
- JSON с конфигом запуска и provenance.

Минимальный объект GeoJSON:

```json
{
  "type": "Feature",
  "geometry": {"type": "Point", "coordinates": [37.0, 44.0]},
  "properties": {
    "detection_id": "S1_20260706_000001",
    "satellite": "sentinel-1",
    "provider": "asf",
    "product_id": "S1A_IW_GRDH_...",
    "acquisition_time": "2026-07-06T08:13:21Z",
    "heading_deg": 82.4,
    "heading_method": "wake_axis",
    "speed_knots": null,
    "speed_method": "not_estimated",
    "confidence": 0.68,
    "validation_status": "unvalidated"
  }
}
```

## 5. Критерии приемки MVP

1. По AOI и времени находятся сцены хотя бы через один fallback-источник.
2. Формируется GeoJSON с кандидатами.
3. Для каждого кандидата сохраняются координаты, время, источник, product id и confidence.
4. При наличии следа рассчитывается направление.
5. Скорость считается только валидным методом или явно не считается.
6. Валидация сохраняет ошибки и статус.
7. Все результаты воспроизводимы по конфигу и product id.
