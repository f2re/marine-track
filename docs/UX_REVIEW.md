# UX/UI аудит Telegram-бота Marine Track

## Цель пользователя

Пользователь хочет быстро понять, есть ли свежий спутниковый снимок по интересующей акватории, запустить детекцию судов и получить пригодные артефакты: обзор, crop судов, GeoJSON/CSV/Parquet и отчет.

## Найденные проблемы

1. `/start` не вел к цели. Пользователь видел список команд и должен был помнить синтаксис.
2. Основной сценарий требовал token: сначала `/dates`, потом кнопка или `/detect token`. Это техническая модель, не пользовательская.
3. Не было главного меню и возврата в безопасную точку навигации.
4. `/status` был техническим dump-ом и не показывал, что мешает работе: AOI, land mask, доступ, output dir.
5. Ошибки не давали следующего действия.
6. Для default AOI не было одной кнопки “найти и обработать”.
7. Результаты детекции не возвращали пользователя в дальнейшие действия.
8. После ручного `/detectbbox` пользователь должен был заново вводить bbox для повторного поиска.
9. Несколько рабочих районов не сохранялись: новый bbox затирал предыдущий.
10. Список сцен обрезался первыми 12 результатами и заставлял пользователя работать с token вручную для остальных сцен.
11. Долгая детекция выглядела как зависание без стадий выполнения.
12. Бот всегда отправлял все артефакты, хотя пользователю иногда нужны только картинки или только файлы.

## Реализованные изменения

### Главное меню

Добавлен модуль `telegram_ui.py` с единым inline-меню:

- `🔎 Найти суда` — основной сценарий: свежая сцена по default AOI и запуск детекции;
- `🕒 Сроки снимков` — показать доступные сцены;
- `↻ Повторить район` — повторить последнюю детекцию по сохраненному bbox;
- `🕒 Сроки района` — показать сроки для последнего bbox;
- `📍 Мои районы` — список сохраненных bbox;
- `📤 Выдача` — режим отправки результата;
- `⚙️ Статус` — человекочитаемый статус;
- `❓ Помощь` — краткая инструкция;
- `🆔 Мой ID` — быстрый id для настройки `TELEGRAM_ADMIN_IDS`.

Кнопки последнего района показываются только если пользователь уже выполнял `/bboxdates` или `/detectbbox`.

### Быстрый сценарий default AOI

Добавлен `detect_default_aoi()`:

```text
default AOI → detection-capable scene search → register scene token → run detection → send outputs
```

Пользователь может нажать `🔎 Найти суда` без ручного ввода bbox/token.

### Сохраненные районы пользователя

`telegram_user_state.py` хранит список `saved_bboxes` per-user в `telegram_user_state.json` внутри `MARINE_TRACK_OUTPUT_DIR`.

Для каждого bbox сохраняются:

```text
id, label, sensor, west, south, east, north, hours, created_at, updated_at, use_count
```

Поведение:

- одинаковые bbox не дублируются, а обновляют существующую запись;
- на пользователя хранится до 10 районов;
- если район один, меню показывает быстрые кнопки `↻ Повторить район` и `🕒 Сроки района`;
- если районов несколько, меню показывает `📍 Мои районы`;
- `/areas` открывает список сохраненных районов с действиями `🔎 Детекция`, `🕒 Сроки`, `🗑 Удалить`.

### Пагинация сцен

Список `/dates` и `/bboxdates` теперь выводится страницами по 6 сцен. В сообщении показываются время, sensor, provider, beam/pol/cloud и короткий product id.

Кнопки `◀️ Назад`, `▶️ Далее` и `стр. X/Y` используют callback по локальному token из `scene_registry.json`. При перелистывании бот перечитывает сохраненный `scenes_json` и не делает новый provider API search. Если registry или файл результата устарел, пользователь получает короткую ошибку: выполнить `/dates` или `/bboxdates` заново.

### Progress states

Долгий detection flow теперь показывает стадии:

```text
1/5 search
2/5 materialize
3/5 detect
4/5 render
5/5 send
```

Стадии `materialize`, `detect` и `render` приходят из `detection_pipeline.py` через progress callback, то есть отражают реальное прохождение pipeline, а не только UI-текст.

### Режим выдачи результата

Добавлен per-user режим выдачи:

```text
🖼 Картинки  overview.png и crop судов
📄 Файлы     GeoJSON, CSV, Parquet, report.json
🧾 Всё       картинки и файлы
```

Режим выбирается через `/output` или кнопку `📤 Выдача` и хранится в `telegram_user_state.json`. После детекции бот отправляет только выбранные артефакты.

### Улучшенные сообщения

- `/start` теперь показывает цель, быстрый путь, текущий default AOI и последний bbox, если он есть.
- `/help` разделяет сценарии “без ручного ввода” и “точное управление командами”.
- `/status` показывает доступ, AOI, lookback, sensor, land mask, shoreline buffer, output dir, последний bbox и режим выдачи.
- Ошибки `/detect`, `/detectbbox` и MaterializationError теперь предлагают вернуться к меню или использовать быстрый сценарий.

### Навигация после результата

После детекции summary-сообщение содержит меню. Пользователь может сразу запустить новый поиск, повторить последний район, открыть сроки, статус, помощь или поменять режим выдачи.

## Текущий рекомендуемый сценарий

1. `/start`
2. При необходимости выбрать `/output` или `📤 Выдача`.
3. `🔎 Найти суда`
4. Дождаться progress states и результата.
5. Для ручной зоны использовать:

```text
/detectbbox sentinel1 west south east north 12
```

После ручного bbox можно дальше использовать кнопки:

```text
↻ Повторить район
🕒 Сроки района
```

Если районов несколько:

```text
/areas
📍 Мои районы → 🔎 Детекция / 🕒 Сроки / 🗑 Удалить
```
