# Калибровка Marine Track через Telegram

## Назначение

Административная калибровка создаёт воспроизводимый human-in-the-loop контур для разметки результатов текущего candidate detector. Она не превращает текущий CFAR/Hough pipeline в подтверждённую ML-модель и не делает `ranking_score` вероятностью судна.

Первая версия решает четыре задачи:

1. собирает положительные и отрицательные ответы по реальным кандидатам;
2. фиксирует ошибки локализации;
3. пересчитывает эмпирические коэффициенты ranking score;
4. сохраняет рекомендации для post-filter параметров без неконтролируемого изменения CFAR и speed proxy.

## Запуск

Интерфейс доступен только пользователям из `TELEGRAM_ADMIN_IDS`:

```text
/calibrate
```

Также кнопка `🧪 Калибровка` появляется в административном меню. Если активного профиля нет, бот:

- отправляет предупреждение администратору при запуске процесса;
- повторяет предупреждение при `/start`;
- отмечает кнопку калибровки символом `⚠️`.

Startup-уведомление доставляется только если администратор ранее открыл личный чат с ботом. Ошибка доставки не останавливает polling.

## UI/UX сценарий

Для очередного неразмеченного кандидата бот строит квадратный crop и делит его на сетку 3×3.

Администратор выбирает:

- `1…9` — клетку с центром корпуса судна;
- `🚫 Судна нет` — ложная цель: шум, берег, платформа, волновой фронт или иной объект;
- `❔ Не уверен` — сохранить ответ, но не использовать его для обучения;
- `⏭ Пропустить` — исключить пример из текущей очереди.

Положение исходного candidate намеренно переносится в псевдослучайную клетку. Поэтому нельзя механически нажимать центральную клетку. Если администратор указывает другую клетку, исходный candidate учитывается как ошибка локализации и отрицательный пример для его собственных признаков.

Задания строятся из завершённых запусков:

```text
MARINE_TRACK_OUTPUT_DIR/detections/*/report.json
```

Для появления новых заданий сначала нужно выполнить детекцию по нескольким сценам и условиям наблюдения.

## Хранение состояния

Все данные находятся в отдельном writable-каталоге:

```text
MARINE_TRACK_OUTPUT_DIR/calibration/
├── labels.jsonl
├── profile.json
├── .calibration.lock
└── tasks/
    ├── <task_id>.json
    └── <task_id>.png
```

`labels.jsonl` является append-only журналом ответов. Каждая запись хранит:

- идентификатор задания и candidate;
- администратора и время ответа;
- выбранную и ожидаемую клетку;
- итоговый label;
- snapshot исходных detector features;
- sensor/provider/product/time provenance.

`profile.json` записывается атомарно и содержит версию схемы, объём выборки, коэффициенты, порог, метрики и рекомендации.

## Эмпирическая модель

Базовая эвристика проекта использует три нормированных признака:

```text
peak_score
clamp(contrast_sigma / 8)
clamp((elongation - 1) / 5)
```

До калибровки применяется исходная линейная формула:

```text
0.50 * peak_score
+ 0.35 * contrast_term
+ 0.15 * shape_term
```

После накопления положительных и отрицательных примеров система обучает небольшую regularized logistic model на этих же трёх признаках. Используется балансировка классов. Decision threshold выбирается по комбинации F1 и balanced accuracy.

По умолчанию профиль активируется только после достижения всех условий:

```dotenv
MARINE_TRACK_CALIBRATION_MIN_LABELS=20
MARINE_TRACK_CALIBRATION_MIN_POSITIVE=5
MARINE_TRACK_CALIBRATION_MIN_NEGATIVE=5
```

Пока условия не выполнены, коэффициенты могут отображаться в интерфейсе, но runtime продолжает использовать исходную эвристику.

Важно: вывод logistic model называется `ranking_score`, а не вероятностью. Показанные precision/recall/F1 относятся только к той же обучающей разметке и не являются научной оценкой качества.

## Параметры

```dotenv
MARINE_TRACK_CALIBRATION_MIN_LABELS=20
MARINE_TRACK_CALIBRATION_MIN_POSITIVE=5
MARINE_TRACK_CALIBRATION_MIN_NEGATIVE=5
MARINE_TRACK_CALIBRATION_CROP_SIZE_PX=768
```

Размер crop ограничивается диапазоном 384–1200 px и приводится к числу, кратному трём.

## Что корректируется автоматически

После активации профиля новые кандидаты получают ranking score по сохранённой эмпирической logistic formula. В metadata записываются:

```text
ranking_score
ranking_score_kind
calibration_profile_id
```

Старое поле `confidence` временно содержит тот же ranking score для обратной совместимости текущей схемы. Оно не должно отображаться как вероятность.

## Что пока не корректируется автоматически

Candidate-only разметка не видит пропущенные суда ниже текущего CFAR threshold. Поэтому по ней нельзя корректно определить:

- `threshold_sigma`;
- размер training/guard window;
- recall полного детектора;
- false alarms на км²;
- wake geometry thresholds;
- Kelvin speed formula и её коэффициенты.

Система вычисляет только консервативные рекомендации для `min_contrast_sigma` и диапазона площади положительных candidates. Они сохраняются в профиле с `applied_automatically=false`.

Автоматически менять CFAR по candidate-only выборке запрещено: это создало бы feedback loop и скрыло бы false negatives.

## Следующие этапы

### Этап 2 — scene-level negative sampling

Нужно генерировать случайные water-only cells и cells вокруг rejected components. Это позволит оценивать ложные тревоги на площадь и подбирать CFAR/post-filter параметры.

### Этап 3 — missed-target annotation

Администратор должен размечать суда на tiles, построенных независимо от текущего detector. Только после этого можно оценивать recall и оптимизировать `threshold_sigma`, training ring и area limits.

### Этап 4 — wake calibration

Отдельный диалог должен размечать:

- наличие/отсутствие следа;
- ось и кормовой сектор;
- arms и continuity;
- неоднозначность курса;
- согласование с AIS.

### Этап 5 — sensor-stratified profiles

Профили должны разделяться как минимум по sensor, processing level, polarization/band, GSD и типу акватории. Нельзя применять один набор коэффициентов одновременно к Sentinel-1 SAR и Sentinel-2 optical.

### Этап 6 — benchmark split

Разметка должна быть разделена по сценам и проходам на train/calibration/test. На test split рассчитываются precision, recall, F1, POD, FAR, CSI, false alarms/km² и localization error с bootstrap confidence intervals.
